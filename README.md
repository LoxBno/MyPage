# 个人数字名片页

## 作品简介
一个简洁美观的个人数字名片网页，展示个人基本信息、介绍、技能与兴趣，可公开访问。

## 页面主要内容
- 姓名、学号、专业
- 个性签名
- 不少于100字自我介绍
- 兴趣标签、技能标签
- 个人头像
- 外部链接
- 整齐排版样式

## 文件结构
MyPage/
├─ index.html    主页
├─ style.css     样式文件
├─ images/       图片资源
└─ README.md     项目说明

## 访问方式
本项目已开启 GitHub Pages，可通过公开链接访问。

## 页面公开访问方式
1.  将项目上传至 GitHub 公开仓库
2.  开启 GitHub Pages 服务
3.  通过生成的公开链接访问

## 体现“软件作品”而非单个页面的核心点
1.  **工程化结构**：采用 HTML（结构）、CSS（样式）、资源（图片）分离的规范架构，符合前端开发标准

3.  **可部署性**：支持公网部署，具备完整的发布流程，可在线访问
4.  **文档完整性**：包含详细的 README 说明文档，符合软件作品的交付规范

## AI辅助开发说明
### 1. 是否使用AI
是，开发过程中使用了AI工具辅助完成项目。

### 2. 使用了哪些工具
豆包，千问，GPT。

### 3. 关键提示词（2-4条）
- 设计一个有个性的个人数字名片页，采用不对称分栏布局，赛博简约风配色，有动态交互效果
- 修复个人数字名片页代码空白问题，排查文件路径和编码错误
- 美化个人数字名片页布局，增加渐变背景、毛玻璃卡片、标签悬浮动效

### 4. AI主要帮助了哪些部分
- 排查并修改了导致页面空白的错误代码；
- 对README进行修改；
- 美化页面布局，设计了不对称分栏结构、赛博风配色方案及动态交互效果。

### 5. 你对AI结果做了哪些修改
主要修改了AI生成的美化后布局细节：调整了分栏的宽度比例、标签的间距和大小，优化了手写体签名的显示位置，适配了本地图片路径的实际情况。

### 6. 你认为AI最不可靠的一点是什么
AI生成的代码有时会忽略实际的文件路径、编码等本地化细节问题，且布局适配的细节（如不同设备的尺寸适配）不够贴合实际使用场景，需要手动验证和调整；此外，AI生成的设计风格可能存在“模板化”倾向，个性化细节需要人工二次优化才能符合预期。
